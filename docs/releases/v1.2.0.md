# Release v1.2.0

Two themes: a smoother app launch, and the new **Residents** platform capability
(verified homeowners / renters / commercial owners, resident-only announcements,
and a resident-aware concierge).

> **Submitting: Balkina AI base app only.** The white-label property apps are
> for pitching now and get submitted per-client when a client signs. The
> Residents capability lives in the same codebase and powers those property
> apps; in the base Balkina binary the resident screens only activate in a
> property build, so they aren't reviewer-reachable — keep the App Store
> "What's New" to the launch improvements (see below).

## Customer app (App Store / Play Store)
- **Smoother launch.** The property app holds the native splash and cross-fades
  straight into the storefront — no more intermediate green/blue loading screen
  or clunky right-to-left transition. The base Balkina app boots the same way.
- **Verify your residence.** In a property app, the account menu now has
  *Verify your residence*: enter the code your property gave you (or that was
  emailed/SMS/WhatsApp'd to you) and you're instantly verified — no approval.
  A resident badge (e.g. "Homeowner · Villa 12") then shows in the account menu.
- **Resident-only announcements.** Campaigns/announcements targeted at residents
  only appear for verified residents.
- **Resident-aware concierge.** The AI concierge greets verified residents and
  prioritises resident perks.
- **Polish.** Fixed the verification "Done" button width on the confirmation
  dialog.

## Property dashboard (Vercel)
- **Residents tab** (formerly "Members"): invite residents individually by
  email / SMS / WhatsApp, or issue shared group codes; see verified residents,
  change type, revoke/restore. Decluttered into action buttons + Residents /
  Invites / Codes tabs.
- **Messages → Residents.** Alongside business email (All / Select / By
  category), you can now push an announcement to residents (All members /
  Homeowners / Renters / Commercial owners).
- **Campaign audience targeting:** All / Residents / a specific resident type.
- Invite emails send under the property's own name (no "via Balkina AI") with
  Download-the-app buttons.

## API / data
- Migrations **060–063**: `property_members`, `property_member_codes`
  (+ email/phone invites), `property_campaigns.audience`,
  `property_messages.recipient_label`.
- New endpoints: member redeem/status, member codes & invites, resident push
  messaging, category-aware business messaging.

## Build & submit (Balkina AI base app)
Run from an authenticated machine (EAS CLI + Apple credentials):

```bash
eas build --profile production --platform ios
eas submit --profile production --platform ios   # ascAppId 6761651423
```

Build numbers auto-increment remotely (`eas.json` → `appVersionSource: remote`);
`version` is 1.2.0. White-label property apps are submitted separately, per
client, when signed.
