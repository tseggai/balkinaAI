# Release v1.2.0

A large release: **Calendar Sync, OTA distribution (Bokun), web booking,
multi-currency, business-type tags**, plus the new **white-label property
platform** (branded storefront, campaigns/events, and **Residents**).

> **The Balkina app is one multi-role binary** — the same app serves customers,
> business owners, and staff by login. Most v1.2 features (Calendar Sync, Bokun,
> settings, staff app, multi-currency) ship in the **base app** and are
> reachable in review. The **white-label property storefront / campaigns /
> Residents** are gated to per-client property builds (`propertySlug`) and are
> *not* reachable in the base binary — so they're kept out of the base App Store
> notes. App Store submission = **base Balkina AI**, build 11 (v1.2.0), already
> uploaded via CI.

## Full changelog

### New features
- **Calendar Sync** — Google Calendar two-way sync; iCal export to Outlook/Apple
  Calendar; import external calendars (Airbnb, Calendly) to block busy times.
- **OTA Distribution (Bokun)** — sync Viator / GetYourGuide / Airbnb Experiences
  bookings (Settings → Integrations → Bokun).
- **Web Booking** — book from `balkina.ai/b/<business>` in the browser, no app
  needed. *(Web, not the iOS app.)*
- **Business Type Tags** — businesses show a type ("Barbershop", "Spa", …) and a
  short description.
- **Multi-Currency Support** — prices display in the business location's currency.
- **Apple & Google Sign-In** — faster account creation. *(Dev builds; verify in
  production before advertising.)*
- **Subcategory Selection** — owners pick a business type from presets or add
  their own.

### UI / UX
- Redesigned tenant settings (Account Settings / Integrations sub-menus).
- Calendar Sync setup simplified (Google one-tap; iCal with collapsible steps).
- Safer account deletion (expand → type "DELETE" → confirm).
- Consistent full-width booking summary card.
- Staff contact adds Email (alongside Call, Text, WhatsApp).
- Cleaner business cards (distance + drive time on one line; type/description
  under the name).
- Edit-mode safety: closing the profile editor discards unsaved changes.
- Smoother native-splash → home launch transition.

### Bug fixes
- Currency showed `$` everywhere → now the location's currency throughout the
  booking flow (services, packages, extras, summary, confirmation).
- Restored the staff app's 4-tab layout and booking-management actions.
- Unavailable days/weekends/full days now show "not available" instead of
  failing at confirmation.
- Fixed double-booking for the "Anyone" staff option.
- Fixed empty contact buttons for web bookings with no phone (email shows when
  available).
- Day-rate / week-rate services show "Book this day" instead of a confusing slot
  grid.

### White-label property platform (gated to per-client builds)
- Branded storefront (collapsing header, categories, Popular/Trending, business
  pages), property-scoped concierge, account drawer.
- Campaigns & events: RSVP, per-guest QR passes, check-in scanner, calendars.
- **Residents**: verify-by-code membership (shared codes + email/SMS/WhatsApp
  invites), resident-only announcements, resident push messaging,
  resident-aware concierge.

### API / data
- Migrations **060–063**: `property_members`, `property_member_codes`
  (+ email/phone invites), `property_campaigns.audience`,
  `property_messages.recipient_label`.
- New endpoints: member redeem/status, member codes & invites, resident push
  messaging, category-aware business messaging.
